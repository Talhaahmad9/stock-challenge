## `[03] DATABASE SCHEMA`

> All tables have **RLS disabled**. Every DB operation goes through the service role client (`createServiceClient`) which uses `@supabase/supabase-js` directly вҖ” not `@supabase/ssr` вҖ” to guarantee the bypass.

### Tables

```sql
-- Identity & Auth
users           (id, username, password_hash, role: 'admin'|'participant', is_active)
sessions        (id, user_id вҶ’ users, token, expires_at)

-- Competition Structure
events          (id, name, status, starting_balance, total_rounds, current_round, created_by вҶ’ users)
game_state      (id, event_id вҶ’ events, status: event_status enum, current_round,
                 timer_remaining, paused_at, last_updated)
rounds          (id, event_id вҶ’ events, round_number, duration_seconds, status, started_at, ended_at)

-- Market Data
stocks          (id, event_id вҶ’ events, symbol, name)
stock_prices    (id, stock_id вҶ’ stocks, round_id вҶ’ rounds, price)

-- Trading
portfolios      (id, user_id вҶ’ users, event_id вҶ’ events, balance)
holdings        (id, portfolio_id вҶ’ portfolios, stock_id вҶ’ stocks, quantity)
trades          (id, user_id вҶ’ users, event_id вҶ’ events, portfolio_id вҶ’ portfolios,
                 stock_id вҶ’ stocks, round_id вҶ’ rounds, type: 'BUY'|'SELL', quantity, price)

-- Audit
admin_actions   (id, event_id вҶ’ events, admin_id вҶ’ users, action, created_at)
```

### `game_state.status` вҖ” PostgreSQL Enum

```sql
CREATE TYPE event_status AS ENUM (
  'IDLE', 'SETUP', 'READY', 'RUNNING',
  'ROUND_ACTIVE', 'ROUND_END', 'GAME_END', 'RESET'
);
```

The `game_state` row is created **automatically** when an event is created. Never needs manual insertion.

### Deletion Cascade Order

When deleting an **event**, FK constraints must be resolved in this order:

```
admin_actions вҶ’ trades вҶ’ holdings вҶ’ portfolios
    вҶ’ stock_prices вҶ’ stocks вҶ’ rounds вҶ’ game_state вҶ’ events
```

When deleting a **user**:

```
sessions вҶ’ trades вҶ’ holdings вҶ’ portfolios вҶ’ users
```

## `[05] SOCKET EVENTS`

The Socket.IO server runs as a standalone Node.js process on Railway. Next.js serverless functions cannot maintain persistent connections вҖ” hence the separation.

### Authentication Flow

```
Participant browser
    в”Ӯ
    в”ңв”Җв–ә GET /api/auth/token  (httpOnly cookie вҶ’ server reads it, returns raw JWT)
    в”Ӯ         в”Ӯ
    в”Ӯ         в””в”Җв”Җ JWT cannot be read from JS directly (httpOnly) вҖ” this endpoint
    в”Ӯ              acts as a secure bridge
    в”Ӯ
    в””в”Җв–ә socket.connect({ auth: { token } })
              в”Ӯ
              в””в”Җв”Җ Socket server validates JWT on 'connection' event
                  Invalid token вҶ’ socket.disconnect()
```

### Server вҶ’ Client Events

| Event | Payload | Description |
|-------|---------|-------------|
| `GAME_START` | вҖ” | Game has started, transition from waiting screen |
| `ROUND_START` | `{ roundNumber, durationSeconds, prices: { [symbol]: price } }` | New round began вҖ” includes all stock prices for the round |
| `ROUND_END` | `{ roundNumber }` | Round timer expired |
| `TIMER_TICK` | `{ remaining }` | Countdown tick вҖ” seconds remaining |
| `GAME_END` | вҖ” | Competition over, show final results |
| `TRADE_EXECUTED` | `{ success, newBalance, newHoldings }` | Confirmation of a trade |
| `FORCE_LOGOUT` | `{ reason }` | Admin-triggered force disconnect |

### Client вҶ’ Server Events

| Event | Payload | Description |
|-------|---------|-------------|
| `JOIN_GAME` | `{ userId, token }` | Join the game room for an event |
| `EXECUTE_TRADE` | `{ type, stockId, quantity, eventId }` | Submit a BUY or SELL order |

### Tab: CONTROL

The primary game operations tab.

- **Event Selector** вҖ” dropdown to choose which event is currently being managed. All actions apply to the selected event.
- **Status Badge** вҖ” live game status (`READY` / `RUNNING` / `ROUND_ACTIVE` / `ROUND_END` / `GAME_END`)
- **Round Indicator** вҖ” `ROUND 2 / 5` etc.
- **Countdown Timer** вҖ” live timer fed by `TIMER_TICK` socket events

Three buttons, each with loading spinner:

| Button | Enabled When | Action |
|--------|-------------|--------|
| `START GAME` | `status === READY` | Transitions to `RUNNING` |
| `START ROUND` | `status === RUNNING` or `ROUND_END` | Transitions to `ROUND_ACTIVE`, starts timer, broadcasts prices |
| `RESET` | Always (requires confirmation modal) | Wipes all trades/holdings/portfolios, back to `READY` |

> END ROUND and PAUSE/RESUME have been intentionally removed. Rounds end on timer expiry only.

### Tab: USERS

Full user lifecycle management for the competition.

- **Bulk Generate** вҖ” enter a count and prefix (e.g. `player`, `30`) вҶ’ creates `player01` through `player30` with random passwords
- **Credentials Table** вҖ” shows username + password for every generated user. Passwords are **masked by default** with a per-row show/hide toggle
- **Session persistence** вҖ” credentials survive page refresh via `sessionStorage` but clear on tab close (intentional вҖ” don't leave passwords exposed)
- **Reset Password** вҖ” generate a new random password for any individual user; new password shown in a modal
- **Disable / Enable** вҖ” toggle a user's `is_active` flag. Disabled users cannot log in
- **Delete Individual** вҖ” cascades: `sessions вҶ’ trades вҶ’ holdings вҶ’ portfolios вҶ’ user`
- **Delete All Participants** вҖ” one-click nuke of all non-admin users. Requires confirmation modal. Same cascade order

### Tab: STOCKS

Configure the market for the selected event.

- **Add Stock** вҖ” enter symbol (e.g. `AAPL`) and name. Stock is saved to `stocks` table for the event
- **Set Round Prices** вҖ” for each stock, enter a price per round (e.g. Round 1: 150, Round 2: 175, Round 3: 160). Saved to `stock_prices` linked to the corresponding `rounds` row
- **Delete Stock** вҖ” removes stock and all associated `stock_prices` rows
- Prices entered here are the ones broadcast to all participants when `START ROUND` is clicked

### Tab: MONITOR

Live operational overview during the competition.

- **Trade Log** вҖ” real-time stream of all trades as they execute, received via `TRADE_LOG` socket event. Shows username, action (BUY/SELL), symbol, quantity, price, timestamp
- **Leaderboard** вҖ” full ranked leaderboard polling every 10 seconds. Shows rank, username, cash balance, portfolio value, total value, P&L with rank-change flash animation

### Tab: EVENTS

Create and manage competition events.

- **Create Event** вҖ” form fields: name, starting balance (вӮЁ), number of rounds, round duration (minutes)
- **Event List** вҖ” shows all events with status badge, round count, starting balance
- **Delete Event** вҖ” full cascade deletion. Button is **disabled** during `ROUND_ACTIVE` or `PAUSED` status to prevent mid-round data corruption

On `ROUND_START` socket event, participants:
